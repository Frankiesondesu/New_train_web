# 途轨 · 铁路旅行地图

一个无需构建工具即可运行的铁路旅行网站原型，包含：

- 交互式中国地图与铁路线路筛选
- 公开行政区划 GeoJSON 地理骨架，在瓦片服务不可用时仍保留地图轮廓
- 支持系统主题识别、手动切换并记忆选择的暗色模式
- 地图瓦片错误率过高时自动切换到稳定的行政区划底图
- 点击线路查看沿途车站和代表车次
- 点击车站查看所在线路、停靠车次及周边景点、风景和住宿
- 车站、线路、车次统一搜索
- 桌面端侧栏与移动端底部详情面板
- 铁路数据使用 JSON 存储，车站与各条线路分文件维护

## 运行

由于浏览器直接打开本地 HTML 时可能限制 `fetch()` 读取 JSON 文件，建议在当前目录运行自带的零依赖静态服务器：

```powershell
node server.mjs
```

然后访问 `http://localhost:8080`。

## 数据

- `data/railway-data.json` 是数据 manifest。
- `data/stations.json` 保存车站、坐标、别名和周边旅行内容。
- `data/routes/*.json` 按线路保存线路信息、代表车次、时刻和别名。
- 代表车次、出发到达站、时刻和历时来自 12306 在 `2026-07-15` 的查询结果。
- 站点坐标、沿途景点、住宿和图片为项目静态整理内容，正式上线前应核对授权、准确性和更新机制。

## GitHub Pages 部署

项目已配置 GitHub Actions 静态部署流程：

- workflow 文件：`.github/workflows/pages.yml`
- 触发方式：推送到 `main` 分支，或在 Actions 页面手动运行
- 部署内容：仓库根目录下的静态文件

首次使用时，在 GitHub 仓库的 `Settings -> Pages` 中将部署来源选择为 `GitHub Actions`。之后每次推送到 `main` 都会自动部署。

地图、字体和图片使用在线资源，需要联网加载。
